 ## 4. 序列级观察

 ### 4.1 `D15_20260119061405`

 - 主问题：以可见性不足为主，具体表现为摩托车遮挡、弯腰姿态、暗背景叠加后导致中等尺度 person 仍持续漏检；不是“人太小”，而是“视觉线索弱”。
 - 代表帧：`D15_20260119061405_frame_0345`、`D15_20260119061405_frame_0346`、`D15_20260119061405_frame_0348`、`D15_20260119061405_frame_0355`
 - 下一步建议：优先围绕“遮挡 + 姿态变化 + 暗背景”补充难例观察，并把那批“本应能学会但难度异常升高”的样本单独复查标注或构图问题。

 ### 4.2 `D15_20260119203927`

 - 主问题：它不是 `D15_20260119061405` 的同质序列。当前更明确的主问题是“近邻重叠 / 局部遮挡导致的中等尺度难例”，其中 `0142` 更偏围栏遮挡，`0143 / 0180` 更偏两人重叠。
 - 代表帧：`D15_20260119203927_frame_0142`、`D15_20260119203927_frame_0143`、`D15_20260119203927_frame_0180`
 - 下一步建议：把它与 `D15_20260119061405` 分开看，单独作为 crowded / overlap 机制复盘的核心序列，继续判断它属于“一个框吃掉两人”“第二个人完全没响应”还是“有框但匹配不过线”。

 ### 4.3 `D02_20260123070624`

 - 主问题：更像“构图 + 远景 + 遮挡”叠加，而不是单纯小目标问题；边界小人只占少数，不是这条序列的主因。
 - 代表帧：`D02_20260123070624_frame_0060`、`D02_20260123070624_frame_0061`
 - 下一步建议：继续把它作为“远景内部小人并非纯尺度问题”的对照序列保留观察，不建议为这条序列单独开边界分支或专门补一类远景内部小人样本。

 ### 4.4 `D02_20260123074836`

 - 主问题：边界远景小人确实存在，但更准确的难点是“贴边 + 远景 + 半身可见”，而不是单一的边界属性；这类样本目前看也没有达到 fullframe 模型完全无能为力的极限。
 - 代表帧：`D02_20260123074836_frame_0022`、`D02_20260123074836_frame_0023`、`D02_20260123074836_frame_0024`
 - 下一步建议：继续把它作为边界类对照序列，不建议因为这批样本单独开实验分支，也不建议把“边界小人”误判成当前 hardest FN 的主因。

 ### 4.5 `D05_20260123074841`

 - 主问题：多人近邻 / 局部遮挡问题，核心不是“完全没看见”，而是多人相互干扰后可见性变差；结合 `D15_20260119203927` 的新结论看，它更适合作为 crowded / overlap 主子类型的辅助对照序列，而不只是孤立的次要异常点。
 - 代表帧：`D05_20260123074841_frame_0026`、`D05_20260123074841_frame_0029`、`D05_20260123074841_frame_0030`
 - 下一步建议：和 `D15_20260119203927` 一起进入 crowded / overlap 机制复盘，但现阶段仍不建议直接跳到 NMS 调参，先确认漏检到底是合框、漏响应还是匹配不过线。

 ## 5. 最终结论

 - 当前 hardest FN 最主要的语义原因：不是“纯边界小人”或“纯极小目标”，而是至少分成两条稳定主类型：
   1. `D15_20260119061405` 代表的“可见性弱型”，核心是遮挡、暗背景、姿态异常叠加后使中等尺度 person 视觉线索不足；
   2. `D15_20260119203927` 与 `D05_20260123074841` 代表的“crowded / overlap 型”，核心是近邻重叠、局部遮挡和多人干扰。
 - 当前更稳的全局判断：`small_boundary_person` 仍不是当前 hardest FN 的主矛盾；边界远景小人存在，但更多是对照子问题，而不是驱动整体失败结构的核心来源。
 - 当前最值得优先做的改进动作：
   1. 把 crowded / overlap 的 5 张代表帧单独拉出来做机制复盘，明确是“一个框吃掉两人”“第二人无响应”还是“有框但匹配不过线”；
   2. 后续讨论或实验设计时，不再把两条 D15 序列混成同一种失败模式，而是分别沿“可见性弱型”和“crowded / overlap 型”两条线推进；
   3. 如果后续要把这批记录用于更正式的复盘台账，再回补 `split`、`gt_index`、`reviewed_by`、`reviewed_at` 等元数据。
 - 当前明确不建议优先做的动作：
   1. 不建议把 `small_boundary_person` 直接当成当前主因；
   2. 不建议仅靠继续放大 `imgsz` 来期待根本改变 hardest FN 结构；
   3. 不建议在现阶段还把 crowded / overlap 当作“很小的边角子问题”，也不建议尚未做机制复盘就直接跳到 NMS 调参。
