
# person fullframe 扩样后的小目标改进方案

## 1. 当前问题定位

当前 `person_fullframe_with_new_labels_baseline` 的 test 指标为：

- Precision `0.9304`
- Recall `0.8552`
- mAP50 `0.9054`
- mAP75 `0.4527`
- mAP50-95 `0.4802`

这组结果说明：

1. 扩样是有效的，模型已经比旧 `person_fullframe_baseline` 更会“发现人”；
2. 但当前主要瓶颈不再是“能不能检出来”，而是“框得够不够准”；
3. 尤其 `mAP50` 明显高于 `mAP50-95`，`mAP75` 也偏低，说明高 IoU 下的框定位精度仍然一般；
4. 对当前任务来说，问题大概率集中在：小目标、远景、遮挡、半身、逆光和密集场景中的框回归误差。

一句话概括当前状态：

> 现在不是完全没学到，而是已经能看到更多 person，但对小目标和复杂样本的 bbox 还不够精确。

## 2. 为什么当前重点要看 `mAP50-95`

如果只看 `mAP50=0.9054`，这版模型会显得很强；但当前更应该盯住：

- `mAP75`
- `mAP50-95`

原因是：

- `mAP50` 更偏向“是否大致检测到”；
- `mAP75` 和 `mAP50-95` 更能反映“小目标框得准不准、边界贴不贴合”。

因此，后续提升方向不应再只围绕“多检出一些人”，而应优先围绕：

- 小目标是否有足够像素；
- 框回归是否足够稳定；
- 难样本是否覆盖充分。

## 3. 解决方案总览

### 3.1 第一优先级：先做输入分辨率单因子对照

这是当前最优先的动作。

建议顺序：

1. 先做 `imgsz=768`
2. 再决定是否继续做 `imgsz=960`

核心目的：

- 给远景人、小人、半身人更多有效像素；
- 验证当前 `mAP50-95` 偏低，是否主要是输入分辨率不足导致。

评估时重点不要只看总分，而要重点看：

- test `mAP50-95`
- test `mAP75`
- 小目标场景下的 FN / bbox 偏移

当前第一步的训练 / 评估命令已写入：

- `backend-train-model/person-train-model/train-docs/person_run_method.md`

### 3.2 第二优先级：补难小目标样本，而不是泛泛扩样

如果 `imgsz=768` 后提升有限，下一步不要先急着换模型，而要先补下面这些类型的样本：

- 远景小人
- 遮挡人
- 半身人
- 逆光 / 背光人
- 密集区域中的小人
- 靠近边缘、容易框偏的小人

优先复查的历史 hard sequences 仍然包括：

- `D15_20260119061405`
- `D15_20260119203927`
- `D02_20260123074836`
- `D02_20260123070624`

重点不是简单“再加更多普通 person”，而是**定向补最容易拉低 `mAP50-95` 的难样本**。

### 3.3 第三优先级：提高定位质量，而不是继续只堆 recall

当前问题已经从“检不出来”转向“框不够准”，因此训练策略应适度往定位精度倾斜。

建议：

- 后期适当减弱过强的数据增强；
- 尽量保留更多原图局部细节；
- 不要只追求更高 recall，要同步看框的贴合度。

这条的意义在于：

- 强增强有时有利于“检出”，
- 但不一定有利于“小目标高 IoU 框回归”。

### 3.4 第四优先级：在前面几步之后，再考虑更高容量模型

如果下面几步都完成后，`mAP50-95` 仍然卡住：

1. `imgsz=768` 已验证；
2. 难小目标样本已补；
3. 训练后期增强策略已调整；

再考虑做更高容量模型对照，例如：

- `yolov8s`

当前不建议一上来就切更大模型，因为那样很容易把：

- 输入分辨率问题
- 样本结构问题
- 模型容量问题

混在一起，结论不清楚。

### 3.5 第五优先级：如果小目标集中在固定区域，再考虑局部裁切 / 切片

如果后续复盘表明：

- 小人长期集中在某些固定区域；
- fullframe 下缩放后人过小；

可以考虑：

- 局部 crop 训练；
- 切片训练 / 切片推理；
- 保留上下文的局部放大方案。

这类方案更适合解决：

- 大画面远景小目标；
- 稠密区域人太小的问题。

## 4. 当前不建议优先做的事

### 4.1 不建议只继续加 epochs

当前更像是：

- 小目标像素不够；
- 难样本覆盖不足；
- bbox 精度不够；

而不是单纯“没训够”。

### 4.2 不建议只看 `mAP50`

`mAP50` 会掩盖定位问题。当前这条线必须把：

- `mAP75`
- `mAP50-95`

作为主观察指标。

### 4.3 不建议一次同时改太多变量

不要在同一轮里同时改：

- `imgsz`
- `batch`
- 模型大小
- 增强策略
- 数据构成

否则后面很难判断到底是哪一个因素带来了收益。

## 5. 推荐执行顺序

当前更稳妥的下一轮顺序建议写成：

1. 先跑 `person_fullframe_with_new_labels` 的 `imgsz=768` 对照；
2. 重点看 `mAP50-95 / mAP75 / 小目标漏检` 是否明显改善；
3. 若有效，再考虑 `imgsz=960`；
4. 若提升仍有限，就转去做难小目标样本补强；
5. 再之后才考虑更高容量模型；
6. 如果问题明显集中在固定区域小目标，再考虑局部裁切 / 切片方案。

## 6. 当前一句话结论

当前 `person_fullframe_with_new_labels_baseline` 的主要问题不是“不会检测 person”，而是**对小目标和复杂场景中的 person，框还不够准**；因此下一步最该做的是：

> 先做 `imgsz=768` 的分辨率对照，再用 hard sample 复盘决定是否继续升到 `960`、补难样本，或再升级到更大模型。
