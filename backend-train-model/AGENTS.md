# backend-train-model 目录说明

本文件作用域覆盖 `backend-train-model/` 目录下的全部代码与文档。

## 当前阶段目标

当前阶段只聚焦 **后端模型训练链路** 的完善与可复现性增强，重点包括：

- 数据入口与默认配置项目化；
- `personcrop` 与当前工服检测方案对齐；
- 训练 / 评估 / 导出报告可追溯；
- 训练产物更便于后续接入现有在线链路。

`inspection-flask/` 与仓库根 `docs/` 仅作为参考来源；除非用户明确要求，否则不要在本轮后端训练任务中修改它们。

## 更新日志强制要求

当你修改 `backend-train-model/` 下任意代码、配置或文档时，必须在同一轮修改中同步更新：

- `backend-train-model/docs/update_log.md`

更新时必须遵守该文件中的“维护约束”，至少写清：

1. 变更来源；
2. 变更总览；
3. 涉及文件；
4. 新增 / 变更配置项；
5. 兼容性注意；
6. 本轮明确不改动的部分。

## 配置修改要求

- 优先修改 `backend-train-model/project_config.json` 或 `backend-train-model/config.py` 的配置入口，不要把新路径、新默认值再次硬编码回训练流程。
- 新增配置项后，要同步检查：
  - CLI 默认值是否已跟随；
  - 训练 / 评估 / 导出报告是否能记录该配置；
  - `docs/update_log.md` 是否已写入对应说明。

## 代码边界

- 不要把当前后端训练链路写成“已完成工人身份识别”的方案。
- 当前默认仍服务于单类 `clothes` 工服检测任务；若扩展到多类，必须明确兼容性影响并写入更新日志。
