# 后端训练完成进度

本文档用于记录 `backend-train-model/` 当前后端训练工作的**真实完成进度**，重点回答：

- 现在已经完成了什么
- 现在还缺什么
- 当前最推荐先推进什么
- 哪些内容只是“数据已准备好”，哪些内容已经真正形成“训练产物”

后续每次完成新的训练、评估、导出、数据补齐或阶段性复盘时，都应同步迭代本文档与 `backend-train-model/docs/update_log.md`。

---

## 1. 使用原则

1. 本文档优先以**真实训练产物**、**真实数据集构建结果**、**真实权重文件**为准。
2. 若某项“文档里已勾选”但仓库中未看到对应产物，应标记为“文档已记录 / 实物待确认”。
3. 若“数据集已建好”但还没有 `best.pt`、评估报告、导出权重，则只能记为“数据准备完成”，不能记为“模型训练完成”。
4. 若后续项目目标、现场约束、误报 / 漏报结论发生变化，应优先尊重真实情况，再更新本文档。

---

## 2. 当前训练主线

当前后端训练并不是“一个模型直接完成全部业务”，而是分阶段推进：

```text
P0：先固化 clothes/workwear baseline
P1：补 person 数据资产
P2：训练并固化 person 模型
P3：启用 personcrop，重训 clothes
P4：离线搭建完整链路（person + clothes + ROI + temporal）
P5：接实时摄像头，做线上联调与复盘
```

当前最稳的阶段性结论是：

- `clothes/workwear` 仍是已完成度最高的训练主线
- `first-train` 的 `best.pt` 仍是当前更稳的 clothes 基线
- `All-train-model` 已经完成 `merged_v1` 一轮闭环，也完成了 `merged_v2` 数据补齐
- 但 `merged_v2` 还没有形成新的训练 / 评估 / 导出产物
- `person` 与 `personcrop` 仍处于未闭环状态

## 6. 当前推荐顺序

### 6.1 训练主线上的最近一步

如果只讨论“后端训练”本身，当前最优先的下一步是：

1. 保持 `first-train` 的 `best.pt` 作为稳定基线
2. 基于 `merged_clothes_v2_full_reviewed` 补出下一轮 clothes 训练闭环
3. 生成对应 train / eval / export 产物
4. 再与 `first-train` 做同口径对比

### 6.2 训练之外必须补的配套资产

与训练结果同样重要的，是同步沉淀：

1. baseline 误报样本清单
2. baseline 漏报样本清单
3. 后续 merged 版本与 baseline 的对比结论

如果这部分不补齐，后续即使继续训练，也会难以判断性能变化究竟来自：

- 数据更好了
- 训练更好了
- split 更碰巧
- 还是现场误报结构根本不在训练层

### 6.3 更后面的主线

在 clothes 第二轮闭环之后，再按顺序推进：

1. 补齐 `person` 数据资产
2. 固化 `person_detect_yolov8.pt`
3. 启用 `personcrop` 重训 clothes
4. 做 `fullframe` vs `personcrop` 对比
5. 接离线链路
6. 接实时摄像头
