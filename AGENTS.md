# AI Agent 仓库说明

本仓库包含一个用于目标检测或相关视觉任务的数据集说明文档，供 Codex、Claude Code 以及其他 AI Agent 在执行以下任务前优先读取：

- 数据集理解
- 标注格式解析
- 训练配置编写
- 数据转换脚本编写
- 数据校验与问题排查

## 必读文件

当任务涉及数据集时，请先读取以下文件：

- `docs/dataset.md`

## 示例文件

在理解数据集格式时，请同时查看以下示例文件：

- 示例图片：`docs/dataset_examples/sample_001.jpg`
- 示例标注：`docs/dataset_examples/sample_001.txt`

说明：

- 默认假设整个数据集与示例文件使用相同的命名规则和标注格式。
- 不要仅根据文件名猜测类别含义、坐标格式或任务类型。
- 如果图片与标注说明存在冲突，以 `docs/dataset.md` 中的文字说明为准。
- 如果只看到图片而没有对应标注文件，图片只能作为视觉参考，不能单独用于推断完整标注规范。
- 如果上述示例文件尚未放入仓库，请先提示用户补充示例文件，再进行依赖数据格式的代码修改。

## Agent 工作要求

- 在修改训练、推理、评估、数据转换、可视化相关代码前，先读取 `docs/dataset.md`。
- 如果需要写数据加载器、标注转换器、校验脚本，先检查示例图片与示例标注是否一一对应。
- 如果文档中标明“所有样本标注方式一致”，可按该假设处理；否则不要擅自泛化。

## 业务 / 方案文档约束

- 仓库中的业务说明、方案设计、检测链路、生产路线类文档，默认用于帮助 Agent 理解项目当前阶段的目标、进度、代码现状和演进方向。
- 这些文档**不是不可质疑的固定真理**，也不必被视为永久不变的最终结论。
- 当真实生产目标、现场约束、当前代码实现、已观察到的误报 / 漏报与现有文档冲突时，应优先尊重真实情况与用户最新说明。
- 在这种情况下，Agent 应明确指出冲突点，提出修订建议；经用户批准后，再更新对应文档。
- 更新业务 / 方案类文档时，优先显式区分：
  - 真实生产目标
  - 当前阶段实现
  - 当前能力边界
  - 升级触发条件
  - 后续演进路线
- 不要把“当前代理实现”直接写成“最终业务终局”。

## Python 环境约定

- 本仓库默认使用 Conda 环境 `yolo_code` 作为 Python 解释器环境。
- 该环境中的 Python 版本为 `3.9.25`。
- 如无特殊说明，Codex、Claude 及其他 AI Agent 在执行 Python 相关命令、编写脚本或给出运行建议时，均应默认基于该环境理解和处理。
