# 中期汇报

## 阶段成果

截至 `2026-04-21`，本项目已不再只是单一的工服检测课题，而是逐步形成了一个面向加油站 / 巡检场景的完整视频智能检测项目。当前项目由四条任务共同组成：工服检测、隔离栏缺失检测、打电话行为检测、吸烟检测；其中工服检测工程化程度最高，另外三条检测已形成较清晰的阶段成果与方法积累。

### 简略版（约 300 字）

截至 `2026-04-21`，项目已形成工服、隔离栏、打电话、吸烟四条检测任务。工服主线已形成稳定基线，其余三项已完成模型对比或视频级原型验证，项目正进入多风险场景联合推进阶段。

### 详细版（约 800-1000 字）

截至 `2026-04-21`，本项目已形成四条相互关联的检测任务：工服检测、隔离栏缺失检测、打电话行为检测、吸烟检测。它们共同服务于“面向加油站 / 巡检场景的多风险视频智能识别”这一完整目标，而不是彼此孤立的几个小实验。其中，工服检测当前工程成熟度最高，另外三条任务则更多承担场景扩展、方法验证和能力储备的作用。

工服检测方面，项目已经形成较完整的训练主线。`backend-train-model` 中的 `train_workwear.py` 已串起数据审计、prepare、train、evaluate、export 与项目级复核，`dataset_tools.py` 支持 `fullframe` 和 `personcrop` 两种数据准备模式，也支持按序列切分和 holdout 对照；配套脚本还能完成多源 `clothes` 数据合并、manifest 生成和 split 管理，说明这条主线已经具备可复现、可追溯的训练工程能力。当前最重要的阶段成果，是已完成 `first-train` 与 merged 路线在统一 holdout 上的公平比较，确认 merged 更优，并将 `clothes_merged_v2_balanced_from_first_holdout_v1` 固定为当前暂定 baseline；该模型在 `75` 张图、`150` 个 GT 框的统一对照集上达到 Precision `0.9797`、Recall `0.9653`、mAP50 `0.9875`、mAP50-95 `0.8042`，同时还生成逐图 FP/FN 清单，实现了“有权重、有指标、有复盘”的阶段闭环。`person` 方向也已完成独立数据资产、运行文档以及 fullframe baseline 和 ROI-aware baseline 的训练评估，prepared 数据集达到 `502` 张图、`1658` 个框，说明工服主线正在从单一 clothes 检测逐步走向 `person -> clothes -> ROI -> temporal` 的完整链路。

除工服检测外，另外三条检测任务也已经形成了可以单独讲述的阶段成果。隔离栏缺失检测方面，`otherMonitor/BarrierMonitor` 已经不是单纯的一个检测权重，而是一个“YOLO 检测器 + ROI + 时序状态机”的告警原型：代码中同时定义了 `barrier_post`、`compliant_barrier`、`idle_barrier`、`car` 四类，并在视频流上完成车辆进入、停稳、等待、隔离栏检查和报警触发的状态流转；仓库中还保留了真实测试视频和导出结果视频，说明这一方向已经推进到“视频级业务事件判断”的原型阶段。打电话检测方面，`otherMonitor/call_runs/calling` 保留了 `YOLOv8s`、`YOLO11n`、`YOLO11s`、`YOLO11m` 等多组 run，对应的 `args.yaml`、`results.csv`、曲线图、混淆矩阵和权重文件较完整；其中较成熟的 run 如 `yolo11m-gpu2`、`yolov8s-gpu2` 的 `mAP50-95` 已达到约 `0.66` 和 `0.65`，证明该任务已经进入模型对比和选型阶段。吸烟检测方面，`otherMonitor/smoke` 不仅保留了训练脚本和本地 Ultralytics 环境，还保留了数据配置、样本子集以及多组 100 epoch 结果归档；其中 `YOLOv8s` run 的 `mAP50-95` 约为 `0.698`，`YOLO11s` 约为 `0.683`，而混合“公共数据集 + 加油站数据集”的 `train5` 达到约 `0.708`，这说明团队已经验证了跨场景迁移和混合训练的可行性。

从整体上看，这四条任务共同说明：项目的能力边界已经不再局限于单一工服检测，而是在逐步形成一个面向加油站 / 巡检场景的多风险智能识别项目。当前仍存在的主要问题是：工服主线中的 `person` 共享默认权重尚未固化，`personcrop` 与 `fullframe` 的公平对照尚未开始；另外三条检测任务虽然已有明显成果，但大多还停留在“阶段归档”和“原型验证”层面，尚未统一到和工服主线同等级的标准化基线体系中。因此，本阶段最准确的结论是：项目已经完成四类检测任务的阶段性铺开，并在工服检测主线上率先形成稳定基线；下一步应在保持多任务项目结构的基础上，继续推进权重固化、链路联调与统一化整理。

## 今后计划

下一阶段的工作，重点还是放在四个检测方向的继续完善和实际验证上。前一阶段已经把工服、隔离栏、打电话、吸烟这几条线陆续做起来了，也积累了不少训练结果、测试视频和阶段材料。接下来最重要的，不是单纯再多跑几轮模型，而是把已经有的结果往前推一步，真正做成能够复现、能够对比、能够解释、也能够放到真实视频里使用的东西。换句话说，后面的任务更像“收口”和“补强”：一方面把前面做过的实验整理清楚，另一方面把还比较薄弱的地方补起来，尽量让每个方向都形成从数据、训练、验证到展示的完整闭环。

工服检测方面，后续准备继续把已经搭好的训练链路做实。现在这一块的基础相对清楚，后面主要做三件事。第一件事是把 `person` 模型的默认权重和使用口径固定下来，把目前几种可选方案做成统一对照，减少后续反复切换带来的时间消耗。第二件事是把 `fullframe` 和 `personcrop` 两条路线认真比完，不只看某一次训练的分数，还要把典型误报、漏报、遮挡场景和视频表现一起放进来判断，看看哪条线更适合当前项目的实际场景。第三件事是把 `person -> ROI -> clothes -> temporal -> alarm` 这一整条链路真正串起来，尽量做到从画面输入到最终提示都能跑通，并能留下复盘证据。除此之外，还会继续整理难样本，例如远距离人员、多人同时出现、动作幅度大、穿着不规范但与背景接近等情况，把这些样例单独抽出来看，作为后续继续优化的依据。

隔离栏缺失检测方面，下一步不是简单保留现有原型，而是把原型里依赖经验调出来的那部分内容慢慢固化下来。当前这条线已经能完成车辆进入、停稳、等待、检查和提醒的大致流程，但后面要解决的是“同样的逻辑换个机位还能不能用”“换个站位是不是还稳定”这类更实际的问题。所以接下来的重点，一是整理不同机位下的 ROI 画法和使用规则，把以前靠临时调试完成的部分变成可重复使用的配置；二是把车辆状态判断、等待时长、触发条件和恢复条件进一步细化，减少只在少量视频上看起来有效、换个视频就不太稳定的情况；三是补做更多连续视频验证，而不是只看若干个导出片段。后面如果条件允许，还准备补一批更接近日常巡检的视频样本，把正常情况和异常情况都保留下来，这样在做汇报时不只是有“报警成功”的例子，也有“为什么没有误报”的说明。

打电话检测方面，后面准备先把任务边界再收一收。现在已有的训练结果能够说明这条线是能往前做的，但还需要把“识别持手机”“识别通话动作”“识别明显违规通话行为”这些概念分得更清楚，不然后面不同 run 放在一起比较时，结论容易发散。下一步会在统一的数据划分下重做一轮规范对比，把已有的几组模型结果放到同一口径下重新看，同时把典型误报样例单独整理出来，比如手部遮挡、抬手动作、拿其他物品靠近脸部这些容易混淆的情况。除了静态指标外，后面还会更关注视频中的连续表现，看模型能不能在几秒钟范围内稳定地判断，而不是偶尔在单帧里碰巧识别到。

吸烟检测方面，后面主要有两个方向要继续推进。一个方向是补场景，尤其是夜间、远距离、侧身、遮挡、小目标这些更容易漏检的情况，目前这部分样本还需要再多一些，才能更接近真实使用时会遇到的问题。另一个方向是继续比较不同训练路线，尤其是公共数据和现场数据混合训练后到底带来了哪些提升，哪些场景有效、哪些场景还不够理想，这些都要通过更细的复盘说明白。考虑到吸烟检测本身在视觉上更容易受到光照、姿态和背景干扰，后续还会把一部分精力放在典型误报复盘上，例如把手放到嘴边、遮挡严重、局部亮点干扰等情况，尽量把“为什么会误报、后面怎么改”说清楚，而不是只保留一个最终分数。

除了四个方向本身，后面还要把共用的内容慢慢补起来。现在仓库里已经有不少实验结果、训练记录和阶段视频，但不同方向的整理方式还不完全一样，后续如果不提前理顺，等到后面集中展示或者准备答辩材料时，工作量会突然变大。所以下一步还会同步整理数据台账、模型命名、结果归档、证据图保存、页面展示和汇报材料模板，把各条线的阶段成果尽量按照类似的方式收起来。整体节奏上，前面先把资料、数据和基线收一遍，中间做实测、复盘和参数调整，后面再把页面展示、演示视频和汇报材料补齐。这样往后推进时，不管是继续开发还是做阶段汇报，都不会只停留在“做过一些实验”的层面，而是能比较完整地说明每个方向做到了哪里、下一步准备怎么继续。

## 经费使用情况和经费安排计划

从目前的使用情况看，项目经费主要还是花在研发验证这一阶段，重点不在大规模设备采购，而是在把四个方向的样本、训练、测试和展示慢慢做起来。前期已经产生的支出，基本都围绕数据整理、样本筛选、标注修正、模型训练、视频导出、结果复盘和文档整理这些内容展开。工服检测这边，投入更多放在 `backend-train-model` 相关的数据准备、训练对比、权重评估和复盘材料整理上；隔离栏缺失检测这边，主要花在视频样本整理、规则调试、状态判断验证和结果视频输出上；打电话和吸烟检测这边，费用更多用在多组模型训练、结果归档、典型样本筛查和阶段展示材料沉淀上。除此之外，还有一部分共用支出，例如存储介质、训练环境维护、演示设备接入、样例视频处理、文档整理和汇报材料制作。整体来看，这样的投入方式和项目当前所处的中期研发阶段是比较吻合的，也符合“先把方案做出来、再逐步固化”的实际推进节奏。

后续经费安排上，建议把重点更多放在样本补充、场景验证、结果整理和统一展示这几件事上，而不是过早把经费压在大批量硬件采购或者一次性包装上。原因比较直接：现阶段项目的关键问题并不是“设备不够多”，而是四个方向都还需要继续补数据、补验证、补实测、补复盘。如果这几部分做得不扎实，后面即使有更好的演示硬件，也很难把整个项目讲得特别完整。所以从性价比看，现阶段更值得投入的，还是数据、测试和整理这几类能够直接影响最终效果的内容。

如果按照后续预算做一个大致拆分，可以参考工服检测约 `20%`、隔离栏缺失检测约 `18%`、打电话检测约 `18%`、吸烟检测约 `18%`、公共部分约 `26%` 的方式来安排。这里把公共部分适当放高一些，主要是考虑到后续很多工作并不是单独服务某一条线，比如统一数据台账、模型命名、结果归档、演示页面、告警信息整理、证据图留存、答辩材料制作，这些都需要持续投入，而且会同时服务四个方向。工服检测经费虽然仍需要保障，但后面更多是做链路联调、方案比较和视频验证，不再只是不断扩展单一训练；隔离栏缺失检测的经费主要用在不同机位下的参数调试、连续视频验证和现场样例补充；打电话检测和吸烟检测的经费，重点是补复杂样本、统一口径重训、整理典型误报漏报案例，以及输出适合展示的视频材料。这样分配下来，既能照顾每条线后续推进的实际需要，也能把共用部分真正搭起来。

从使用内容上看，后续经费可以再分成几类去理解。第一类是样本相关支出，包括外场采样、视频截取、样本筛选、标签检查和必要的人工整理，这一部分对四个方向都很关键，因为后面很多误报漏报问题最后都会回到样本质量和样本覆盖上。第二类是训练和验证相关支出，主要包括环境维护、训练试验、结果保存、视频验证和多轮对比，这部分的特点是持续性强，不是一笔投入就能完全结束。第三类是展示和收口相关支出，包括结果图整理、演示视频制作、页面展示、汇报文档排版和答辩材料补充，这一块虽然常常被放在后面，但如果完全不提前预留，最后阶段很容易出现技术结果不少、材料却拼不起来的问题。第四类是一定比例的机动支出，用来应对后续外场补采、模型返工、设备临时维护或者新增展示需求，避免预算过于死板，影响推进节奏。

从时间安排上看，前一段经费更适合先投到数据清理、标签检查和样本补充上，把基础打稳；中间一段经费重点投到四个方向的联调、实测和连续视频复盘上，把模型结果真正放进场景里去看；最后一段经费再集中投到成果收口和展示材料完善上，包括演示视频、界面整理、典型案例图和汇报文档等内容。这样安排的好处是，前中后各阶段的花费都有明确目标，不会出现前面把钱花得太快、后面反而缺少关键验证和展示经费的情况。

总体来说，后续经费安排的思路应当尽量稳一点、实一点，先保证每个方向都能把样本、训练、验证和展示这些基本环节补齐，再逐步去做更大的扩展。等后续批复金额明确之后，可以在上述比例框架下进一步换算成具体数值，并结合实际推进情况做小幅调整。但无论金额大小，优先顺序都不宜改变，也就是先保数据和验证，再保联调和展示，最后再考虑额外扩展。这样更符合项目当前阶段的真实需要，也更有利于后续顺利完成中后期汇报、验收和成果展示。

## 存在的问题及改进方案

从现在的进展来看，项目最大的问题并不是“没有成果”，而是“成果已经有了，但还没有完全收成一套清楚、稳定、方便展示的东西”。四个方向目前都已经做出了阶段结果，只是成熟程度还不太一样。工服检测这边，训练链路和对照关系相对更清楚一些，但完整链路还需要进一步跑顺；隔离栏缺失检测已经把视频原型做出来了，不过在不同机位、不同站位下的稳定性还需要继续验证；打电话和吸烟检测都积累了多组训练结果，也能说明方向是可行的，但还需要把任务边界、数据口径和误报漏报复盘做得更扎实。也就是说，现在的问题不是“有没有做”，而是“做出来的内容还不够整齐、不够统一、不够方便后面继续讲和继续用”。

第一个比较明显的问题，是资料和实验口径还不够统一。现在仓库里已经有不少 run、结果图、视频和阶段说明，但不同方向的整理方式并不完全一样，有些按训练轮次归档，有些按模型版本归档，有些更偏向保存视频效果，有些则更偏向保留分数和曲线。这样做在前期探索阶段问题不大，但到了中后期，如果还延续这种方式，就很容易出现材料分散、结论重复、同一件事很难横向比较的问题。后面的改进办法，首先就是把四个方向的数据来源、标签定义、训练集与验证集划分、关键 run、阶段结论和典型样例统一成较固定的记录方式，至少做到后面再回头看时，能够清楚知道“这次实验是怎么做的、和上一轮相比差在哪、为什么保留这个结果”。

第二个问题，是不少结果还停留在“模型指标可看”的层面，距离“真实视频里稳定可用”还有一步。工服检测后续需要继续把 `person`、ROI、工服识别和时序判断这一整条链路串起来，不然即使单个模型的分数不错，最后也可能在连续视频里表现不稳定。隔离栏缺失检测也存在类似情况，现在原型已经能说明思路是可行的，但如果换一段视频、换一个站位或者换一个机位，稳定性到底怎样，还需要更多连续样本来验证。打电话和吸烟检测的问题则更集中在行为确认这一层，单帧里识别到目标并不等于视频里就能稳定判断违规行为，特别是在手部遮挡、动作相似、目标较小或者光照不好的情况下，误报和漏报都可能明显增多。所以后面的改进不能只盯着训练结果表看，还得把更多精力放到连续视频复盘、误报漏报统计和典型案例分析上。

第三个问题，是复杂场景样本还不够充分。这个问题四个方向都存在，只是表现形式不同。工服检测里，远距离、小目标、多人同框、姿态变化大这几类情况还需要继续补；隔离栏缺失检测里，不同车辆位置、不同摄像头角度、不同天气和光照条件下的表现还需要再多做样本；打电话检测里，手部靠近脸部但不是打电话、拿其他物品贴近耳侧、动作被遮挡这些情况仍然容易造成干扰；吸烟检测里，夜间场景、背光、侧身、远距离、小目标和短时动作的识别难度都比较高。后面的改进方向很明确，就是继续把这些容易出问题的场景单独整理出来，补采也好，复盘也好，至少要让模型在这些场景上的表现被看见，而不是只保留那些相对理想的视频片段。

第四个问题，是目前的项目材料虽然越来越多，但“收口”还不够。简单说，就是做过的东西不少，真正整理成能够直接拿去汇报、答辩、演示的内容还不够成体系。有些实验做完之后，结果是保留下来了，但命名不够统一；有些视频效果不错，但缺少对应的背景说明；有些阶段结论已经能写出来，但没有及时和训练结果、样例图、误报漏报案例放到一起。这会带来一个很实际的影响，就是越到后面，回忆和整理的成本越高，容易出现“明明做过，却一下子找不到最合适证据”的情况。对此，后面需要同步建立阶段资产清单，把权重、数据、结果表、样例图、演示视频、问题清单和结论说明尽量成套归档，不要等到汇报前再临时拼接。

从改进方案上看，比较实际的做法还是分阶段推进。第一步先把资料收口，把命名、目录、版本、结果表和典型案例尽量统一起来，让每个方向都能快速说清楚当前做到哪一步、下一步准备补什么。第二步再针对四个方向分别做一轮补强：工服检测重点补完整链路和不同方案的对照，隔离栏缺失重点补不同机位下的参数和恢复逻辑，打电话检测重点补任务口径和典型误报复盘，吸烟检测重点补夜间、遮挡、远距离等难样本。第三步是在前面这些工作基础上，把页面展示、证据图保存、阶段说明和演示视频再统一起来，尽量做到技术结果和汇报材料同步推进。这样往后做，一方面可以让项目的真实进展更清楚，另一方面也能减少后期集中整理时的重复劳动。整体来看，项目现在的问题并不算方向性错误，更多还是中期阶段常见的“结果有了、整理还差一点、验证还差一步”。只要后面把资料、验证和展示这几块持续补上，整项工作的完整度和可信度还会继续提升。
