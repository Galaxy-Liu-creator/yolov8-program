# Docs 目录说明

本文件作用域覆盖 `docs/` 目录下的全部文档。

## 业务 / 方案文档约束

- `docs/` 中的业务说明、方案设计、检测链路、生产路线类文档，默认用于帮助 Agent 理解：
  - 当前阶段进度
  - 项目目标
  - 代码现状
  - 计划中的演进方向
- 这些文档**不是不可质疑的固定真理**，也不必被视为永久不变的最终结论。

## 与真实生产冲突时的处理原则

当以下信息与现有文档冲突时，应优先尊重真实情况：

- 真实生产目标
- 现场部署约束
- 当前代码实现
- 已观察到的误报 / 漏报
- 用户当前最新说明

在这种情况下：

- Agent 应明确指出冲突点
- Agent 可以对现有文档提出质疑与修订建议
- 经用户批准后，再修改对应文档

## 写作要求

更新业务 / 方案类文档时，优先显式拆分为：

1. 真实生产目标
2. 当前阶段实现
3. 当前能力边界
4. 升级触发条件
5. 后续演进路线

不要把“当前代理实现”直接写成“最终业务终局”。
