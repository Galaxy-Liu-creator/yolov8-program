# inspection-flask 目录说明

本文件作用域覆盖 `inspection-flask/` 目录下的全部代码与说明文件。

## 生产目标优先原则

`inspection-flask` 是实际在线检测与告警链路的实现目录。  
当业务方案、规则说明、历史文档与真实生产目标不一致时，应优先考虑：

- 真实生产要检测什么
- 当前代码实际上实现了什么
- 当前方案还缺少什么能力

## 文档不是写死的

与 `inspection-flask` 相关的业务 / 架构 / 检测链路文档，主要用于记录阶段性方案与实现状态，不应被当成不可修改的最终定论。

如果发现以下情况：

- 文档把过渡方案写成了最终方案
- 文档表述超过了当前代码真实能力
- 文档与生产目标、现场约束、最新需求冲突

Agent 应：

- 明确指出问题
- 提出更符合真实生产的修订建议
- 在获得用户批准后，同步更新文档

## 术语使用要求

在 `inspection-flask` 相关代码和文档中，应尽量准确区分以下概念：

- `ROI 内人员 / 作业区人员`
- `候选作业人员 / 候选员工`
- `已完成身份识别的工人`

如果当前实现仍然只是基于 `person + ROI + track + workwear + temporal` 的代理判断，就不要把它写成“已完成工人身份识别”。
